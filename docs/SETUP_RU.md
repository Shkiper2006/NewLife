# Подробное руководство по установке и запуску «Жизнь с Нуля»

Это руководство написано **максимально просто, как для новичка**. Если вы никогда раньше не запускали проекты на Node.js, React, Vite или WebSocket — ничего страшного. Ниже есть пошаговая инструкция.

---

## 1. Что вы устанавливаете

В этом репозитории две основные части:

1. **Клиент игры** — браузерная 3D-часть на:
   - TypeScript
   - Vite
   - React
   - Three.js
   - @react-three/fiber
   - @react-three/drei
   - @react-three/rapier
   - zustand

2. **Кооп-сервер** — маленький локальный сервер на:
   - Node.js
   - WebSocket (`ws`)

Когда вы запускаете проект в режиме разработки, у вас стартуют:

- **frontend**: страница игры в браузере;
- **backend**: WebSocket-сервер для кооператива.

---

## 2. Что нужно установить заранее

Перед запуском проекта у вас должны быть установлены:

- **Git** — чтобы скачать проект;
- **Node.js** — чтобы запускать JavaScript/TypeScript-проект;
- **npm** — менеджер пакетов, обычно идёт вместе с Node.js.

### Рекомендуемая версия Node.js

Лучше всего использовать:

- **Node.js 20 LTS** или новее.

Проверить версии можно командами:

```bash
node -v
npm -v
git --version
```

Если команды выводят номера версий — всё установлено.

## 3. Установка Git

### Windows

1. Откройте сайт: `https://git-scm.com/download/win`
2. Скачайте установщик.
3. Установите Git, в большинстве случаев можно просто нажимать **Next**.
4. После установки откройте:
   - **Git Bash**, или
   - **PowerShell**, или
   - **Windows Terminal**.

Проверьте:

```bash
git --version
```

### macOS

Вариант 1 — через официальный установщик:

- скачайте с `https://git-scm.com/download/mac`

Вариант 2 — через Homebrew:

```bash
brew install git
```

Проверка:

```bash
git --version
```

### Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install git -y
```

Проверка:

```bash
git --version
```

## 4. Установка Node.js и npm

### Windows

1. Откройте сайт: `https://nodejs.org/`
2. Скачайте **LTS-версию**.
3. Запустите установщик.
4. Во время установки оставьте включённым пункт добавления Node.js в `PATH`, если он есть.
5. После установки **закройте и снова откройте терминал**.

Проверьте:

```bash
node -v
npm -v
```

### macOS

#### Вариант A — официальный установщик

1. Перейдите на `https://nodejs.org/`
2. Скачайте **LTS**
3. Установите как обычную программу.

#### Вариант B — через Homebrew

```bash
brew install node
```

Проверка:

```bash
node -v
npm -v
```

### Linux (Ubuntu/Debian)

Самый простой способ — через `nvm` или через пакетный менеджер. Для новичка удобнее `nvm`, потому что потом легче менять версии Node.js.

#### Вариант A — через nvm

Установка `nvm`:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```

После этого перезапустите терминал и установите Node.js:

```bash
nvm install 20
nvm use 20
```

Проверка:

```bash
node -v
npm -v
```

#### Вариант B — через apt

```bash
sudo apt update
sudo apt install nodejs npm -y
```

Но через `apt` версия может быть старой, поэтому вариант с `nvm` предпочтительнее.

## 7. Что означают скрипты в package.json

В проекте используются следующие команды:

### Запуск разработки

```bash
npm run dev
```

Что делает эта команда:

- запускает **Vite dev server** для клиента;
- запускает **Node.js WebSocket сервер** для коопа;
- позволяет сразу открыть игру в браузере и подключить второго игрока.

### Отдельный запуск клиента

```bash
npm run dev:client
```

Запускает только клиентскую часть.

### Отдельный запуск сервера

```bash
npm run dev:server
```

Запускает только WebSocket-сервер кооператива.

### Проверка типов TypeScript

```bash
npm run check
```

Проверяет TypeScript-код на ошибки типов.

### Диагностика установки

```bash
npm run doctor
```

Проверяет, что вы находитесь в правильной папке проекта и что ключевые зависимости действительно установлены в `node_modules`.

### Production-сборка

```bash
npm run build
```

Собирает production-версию клиента.

### Локальный просмотр production-сборки

```bash
npm run preview
```

Запускает просмотр уже собранной версии.

## 8. Первый запуск проекта — пошагово

Ниже самый простой сценарий для первого запуска.

### Шаг 1. Откройте терминал в папке проекта

```bash
cd NewLife
```

### Шаг 2. Установите зависимости



### Шаг 5. Откройте игру в браузере


Чтобы проверить кооп на одной машине:

1. Откройте первую вкладку: `http://localhost:5173`
2. Откройте вторую вкладку: `http://localhost:5173`
3. Обе вкладки подключатся к одному локальному серверу `ws://localhost:2567`

Теперь можно проверить синхронизацию:

- движения второго игрока;
- подбор предметов;
- установку укрытия.

## 10. Как играть в MVP после запуска

Когда игра откроется в браузере:

### Управление

- `W`, `A`, `S`, `D` или стрелки — ползти
- `E` — подобрать предмет рядом
- `1` — скрафтить верёвку
- `2` — скрафтить простое укрытие
- `Q` — поставить укрытие
- `F` — использовать бутылочку с молоком

### Цель MVP

1. Соберите **2 тряпки**.
2. Нажмите `1`, чтобы получить **верёвку**.
3. Соберите **2 единицы простого мусора/материалов**.
4. Нажмите `2`, чтобы скрафтить **простое укрытие**.
5. Подползите к **жёлтой зоне** у выхода.
6. Нажмите `Q`, чтобы положить укрытие на опасный участок.
7. Избегайте крысы.
8. Доползите до **голубой метки** у пролома наружу.

## 11. Что делать, если `npm install` не работает

Ниже самые частые проблемы.

### Ошибка: `npm: command not found`

Это значит, что Node.js / npm не установлены или не добавлены в `PATH`.

Что делать:

1. Установите Node.js.
2. Закройте терминал.
3. Откройте терминал заново.
4. Проверьте:

```bash
node -v
npm -v
```

### Ошибка прав доступа (`EACCES`, `permission denied`)

#### Windows

- откройте PowerShell / Windows Terminal **от имени администратора**;
- либо установите проект в папку пользователя, например в `Documents`.

#### macOS / Linux

Не используйте `sudo npm install` внутри проекта без необходимости.

Лучше:

- работать из домашней папки пользователя;
- использовать `nvm` для установки Node.js.

### Ошибка сети / прокси / блокировка реестра

Если `npm install` не скачивает пакеты:

1. Проверьте интернет.
2. Проверьте доступ к `https://registry.npmjs.org/`
3. Если вы в корпоративной сети, может понадобиться прокси.
4. Проверьте текущую настройку npm:

```bash
npm config list
```

Если нужен прокси, его можно настроить так:

```bash
npm config set proxy http://YOUR_PROXY:PORT
npm config set https-proxy http://YOUR_PROXY:PORT
```

Если прокси не нужен, а он почему-то прописан, можно удалить настройки:

```bash
npm config delete proxy
npm config delete https-proxy
```

### Ошибка старой версии Node.js

Проверьте:

```bash
node -v
```

Если версия слишком старая, установите **Node.js 20 LTS** или новее.

### Ситуация: `npm install` пишет `up to date, audited 1 package`

Обычно это значит, что npm поставил **не зависимости проекта**, а почти ничего. Самые частые причины:

1. Команда запущена **не в папке `NewLife`**.
2. У вас остались старые `node_modules` или старый `package-lock.json`.
3. Установка оборвалась или npm работает через проблемный прокси/кэш.

Сначала проверьте, где вы находитесь:

```bash
pwd
cat package.json
```

В `package.json` должны быть зависимости вроде `react`, `three`, `vite`, `ws`, `zustand`.

Дальше сделайте чистую переустановку:

#### macOS / Linux

```bash
rm -rf node_modules package-lock.json
npm install
npm run doctor
```

#### Windows PowerShell

```powershell
Remove-Item -Recurse -Force node_modules
Remove-Item -Force package-lock.json -ErrorAction SilentlyContinue
npm install
npm run doctor
```

Если `npm run doctor` покажет, что пакеты не установлены, проблема уже не в проекте, а в окружении npm: прокси, сеть, права доступа или блокировка реестра.

## 12. Что делать, если игра не открывается в браузере

### Ситуация: сервер запустился, но страница пустая

Проверьте:

1. Есть ли ошибки в консоли браузера (`F12` -> Console).
2. Запущен ли клиент через `npm run dev` или `npm run dev:client`.
3. Не занят ли порт `5173` другой программой.

### Ситуация: WebSocket не подключается

Проверьте:

1. Запущен ли сервер командой:

```bash
npm run dev:server
```

2. Есть ли в логах сообщение о запуске `ws://localhost:2567`
3. Не блокирует ли соединение firewall / антивирус.

### Ситуация: второй игрок не появляется

Проверьте:

1. Что открыты **две вкладки** или два окна браузера.
2. Что обе вкладки открыты по одному адресу, например:

```text
http://localhost:5173
```

3. Что обе вкладки подключаются к одному и тому же `ws://localhost:2567`
